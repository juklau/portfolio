MediaStock

Application web de gestion de matériel

Contexte

Le projet MédiaStock a été réalisé dans le cadre du BTS SIO, option SLAM, au cours de la deuxième année de formation.
Il s'agit d'une application web de gestion de matériel multimédia développée pour un établissement scolaire. Avant sa conception, la gestion des prêts de matériel (casques, micros, câbles, transmetteurs, multiprises...) se faisait à l'aide de fiches papier, entraînant un manque de traçabilité et une perte de temps pour le personnel administratif.

MédiaStock a donc été pensé comme une solution numérique complète, permettant de remplacer le système manuel par une interface intuitive, responsive et connectée à une base de données centralisée. Le projet a été réalisé individuellement, avec un déploiement sur serveur local et serveur SISR via Docker.

Besoins

L'objectif principal du projet est de simplifier et automatiser la gestion des prêts et retours de matériel au sein de l'école.

L'application devait permettre à l'équipe administrative de :

  • ◈ gérer l'inventaire complet du matériel (ajout, suppression, modification, archivage),
  • ◈ suivre les prêts et restitutions en temps réel,
  • ◈ identifier chaque matériel grâce à un QR code unique,
  • ◈ effectuer les opérations de prêt directement depuis un smartphone.

Le besoin exprimé portait aussi sur la traçabilité (historique des prêts, retard, état du matériel) et la sécurisation des accès via une authentification administrateur.
Le système devait être rapide à déployer, léger, et fonctionner aussi bien en local qu'à distance pour les tests sur le réseau de l'école

Environnement technologique

Le projet s'appuie sur une architecture trois-tiers (présentation, logique métier, données) et sur des technologies modernes respectant les standards du web :

Backend

  • ◈ PHP 8.2 (procédural) - logique métier et gestion des sessions.
  • ◈ MySQL 8.0 - base de données relationnelle (tables : item, prêt, emprunteur, admininstrateur …).
  • ◈ PDO (PHP Data Objects) - sécurité des requêtes SQL (préparées).
  • ◈ Docker / Docker Compose - conteneurisation complète avec services Apache, MySQL et PhpMyAdmin

Frontend

  • ◈ HTML5 / CSS3 / Bootstrap 5 - structure et mise en forme responsive.
  • ◈ JavaScript (ES6) - interactions dynamiques et scan des QR codes.
  • ◈ Flatpickr - gestion intuitive des dates de prêt et de retour.
  • ◈ Font Awesome - icônes d'interface.

Outils et environnement

  • ◈ VS Code - environnement de développement.
  • ◈ Git / GitHub - versionnage et hébergement du code.
  • ◈ Figma - conception des maquettes UI/UX.
  • ◈ Smartphone Android - tests des fonctionnalités de scan via caméra.

L'ensemble a été conçu pour fonctionner en local (via WAMP/Docker) et sur un serveur SISR distant, garantissant la portabilité et la cohérence de l'environnement.

Réalisation

Le développement de MédiaStock sest déroulé sur six semaines, selon une approche itérative et incrémentale, en suivant un planning hebdomadaire structuré comprenant les phases de définition, développement, tests et déploiement.

Structure de l'application

Architecture PHP procédurale, organisée en dossiers distincts :

  • ◈ /public/api/ : endpoints pour la gestion du matériel et des prêts (additem.php, updateitem.php, cloturepret.php ...).
  • ◈ /config/ : fichiers de connexion et configuration de la base (database.php).
  • ◈ /src/ : classes métiers (Item, Pret, Emprunteur, Administrateur, Categorie, Formation).
  • ◈ /sql/ : scripts d'initialisation et de test de la base (init.sql).

Fonctionnalités principales

  • ◈ CRUD complet du matériel (ajout, modification, suppression, archivage).
  • ◈ Génération automatique de QR codes lors de l'ajout d'un nouvel item.
  • ◈ Scan via smartphone redirigeant automatiquement vers la fiche du matériel.
  • ◈ Gestion des prêts et retours avec mise à jour en temps réel de la disponibilité.
  • ◈ Authentification administrateur avec mot de passe haché et session PHP.
  • ◈ Interface responsive permettant l'utilisation sur ordinateur et mobile.

Déploiement

Le projet a été déployé en local via Docker (http://localhost:8080) puis transféré sur le serveur SISR mutualisé pour la phase finale de test.
Une documentation complète (technique + utilisateur) a accompagné la livraison, incluant le script SQL, les captures d'écran, le schéma MCD/MLD et le guide d'installation.

Bilan

Le projet MediaStock a permis de concevoir une application web mobile-first dédiée à la gestion des prêts de matériel dans un établissement scolaire.
Partant d'un système entièrement manuel basé sur des fiches papier, le projet a introduit une solution numérique complète, sécurisée et ergonomique, capable de gérer l'inventaire, les prêts, les restitutions, et les retards.

Grâce à une architecture trois-tiers (frontend, backend, base de données), conteneurisée avec Docker, et à une approche itérative du développement, chaque fonctionnalité a été pensée pour répondre aux besoins réels de l'équipe administrative.
Le système de QR codes intégré a considérablement simplifié les opérations terrain, tout en garantissant une traçabilité complète.

Ce projet m'a permis de mobiliser mes compétences en conception d'architecture, développement PHP/MySQL, sécurité web, gestion de projet agile, et prototypage d'interface avec Figma. Il représente une solution robuste, évolutive et directement exploitable en contexte réel.

Perspectives

Plusieurs pistes d'amélioration ont été identifiées pour une version ultérieure de l'application :

Évolutions fonctionnelles

  • ◈ Ajout d'un module statistique pour visualiser le taux d'utilisation du matériel et la durée moyenne des prêts.
  • ◈ Intégration d'une gestion multi-comptes avec journal d'audit pour tracer les actions des administrateurs.
  • ◈ Création d'un mode inventaire permettant de comparer le stock physique et théorique par scan successif des QR codes.

Améliorations techniques

  • ◈ Migration progressive vers un framework PHP moderne (Laravel) ou JavaScript (React/Node.js) pour une meilleure extensibilité.
  • ◈ Mise en place d'un hébergement cloud sécurisé (AWS, Azure ou GCP).
  • ◈ Intégration d'API externes pour la synchronisation avec la base de données de l'établissement.

Pérennisation du projet

  • ◈ Documentation continue et formation des utilisateurs internes.
  • ◈ Automatisation de la sauvegarde et de la supervision (monitoring système, alertes de panne).
  • ◈ Ouverture potentielle du projet en open-source éducatif pour favoriser la collaboration entre établissements.
